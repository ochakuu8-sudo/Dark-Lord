# 指示書：コモンレシピの固定素材スロット導入＋逆L追加

## 概要

`src/game/config.ts` の `ALL_RECIPES` 内コモンレシピのパターンを変更する。
全スロットがワイルドカード（`9`）の現状から、**固定素材スロット（`0`=骨, `1`=肉, `2`=霊）を導入**し、単一素材では全レシピが揃わない設計にする。
また、**逆Lテトロミノ（J4）のコモンレシピを1種新規追加**する。

## 設計ルール

- **3マスレシピ（3種）**：固定素材1種×2マス ＋ ？×1マス
- **4マスレシピ（7種）**：固定素材1種×3マス ＋ ？×1マス
- 4マスレシピは**骨・肉・霊それぞれ2つずつ**割り当て、残り1つ（逆L）は自由
- ？ はワイルドカード（`9`）。バリアント決定に使用
- · は空きマス（`-1`）

## 新規追加：ガーゴイル

レシピだけでなく、以下も追加が必要：

- `ALL_RECIPES` に逆L（J4）レシピを追加（`id: "gargoyle"`, `rarity: "common"`）
- `resultMap` に3素材のバリアントを設定（骨→ガーゴイル骨、肉→ガーゴイル肉、霊→ガーゴイル霊）
- `UNIT_STATS` に3バリアントのステータスを追加
- ステータスは既存コモン4マスユニット（アーチャー・オーク等）を参考に、同程度の強さに設定

## 注意事項

- 変更対象は `pattern` フィールドのみ。`id`, `name`, `rarity`, `reward`, `resultMap` は既存レシピでは変更しない
- レアレシピ（necromancer, minotaur, ghoul）は変更しない
- `findMatches()` のマッチングロジックは変更不要（既に固定値とワイルドカードを区別して処理済み）
- 回転マッチングも既存の `precomputePatterns` で自動対応
- `blockPool` 生成（`initPool`）も固定素材を自動でプールに反映するため変更不要
