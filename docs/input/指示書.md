# 実装指示：パッシブ効果タグシステムの完全実装

あなたはローグライク×タワーディフェンスゲーム「Dark-Lord」の開発を担当するコーディングAIです。
以下の設計ドキュメントに基づいて、パッシブ効果タグシステムとそれに紐づく全魔物バリアントの効果を実装してください。

---

## 1. 既存コードベースの構造

本プロジェクトは React + TypeScript + Vite で構築されています。
ゲームロジックは src/game/ 配下の以下3ファイルに集約されています。

### src/game/entities.ts

    export interface PassiveAbility {
      type: string;      // パッシブの種類（例: 'REFLECT', 'POISON' 等）
      value?: number;    // 効果量（ダメージ、割合、スタック数など）
      range?: number;    // 効果範囲（px）
      cooldown?: number; // クールダウン（frames）
    }

    export interface EntityState {
      // ...基本ステータス省略...
      passiveAbilities?: PassiveAbility[];
      // ランタイムフィールド（状態異常スタック等）
      poisonedFrames?: number;
      stunFrames?: number;
      stealthActive?: boolean;
      // ...他の一時フィールド...
    }

ユニットごとのステータスは UNIT_STATS に定義されており、passiveAbilities 配列で効果を持たせます。

### src/game/phrases.ts

定型句タグシステム。パッシブアビリティをトリガー・効果・対象・状態のタグに分解し、レリック判定に使います。

    export const PASSIVE_PHRASE_TAGS: Record<string, Phrase[]> = {
      'REFLECT': ['被弾時', '反射', '攻撃者'],
      'POISON':  ['命中時', '付与', '毒', '対象'],
      // ...
    };

### src/game/config.ts

レシピ定義（ALL_RECIPES）、レリック定義（RELICS）、盤面定数など。
各レシピの resultMap でワイルドスロットの素材に応じたバリアントIDを指定します。

## 2. タグ一覧と効果仕様

以下のタグを **すべて** ゲームロジックとして実装してください。
既に実装済みのものは仕様と一致しているか確認し、未実装のものは新規追加してください。

### 攻撃形状タグ

| タグ | 効果仕様 |
|------|----------|
| 爆発 | 円形の単発範囲攻撃を発生させる |
| 瘴気 | 円形の連続ダメージ領域を作る |
| 跳弾 | 攻撃が近くの別敵へ連鎖する |
| 貫通 | 弾や攻撃が直線上の複数体をまとめて貫く |
| 発射 | 投射物を10°扇状に複数放つ |
| レーザー | 一定時間残る多段攻撃 |

### 状態異常タグ（スタック制）

| タグ | 効果仕様 |
|------|----------|
| 出血 | スタックの値だけ受けるダメージが定数で増える |
| 毒 | スタック量が現体力の値を上回るとスタック分固定ダメージ |
| 炎上 | スタック量の値だけドットダメージを与える。ドットダメージごとにスタック数が半減する |
| 凍結 | 移動速度と攻撃速度をスタック量%で下げる |
| 呪い | 被ダメージを割合増加 |

### 特殊効果タグ

| タグ | 効果仕様 |
|------|----------|
| クリティカル | 効果ごとに確率で発動し、発動時特殊効果をもつ（確率判定はヒット時） |
| 充填 | 時間経過でスタックが増え、固有効果を発動する。0.1秒で1スタック獲得 |
| 追撃 | 条件時CD関係なく攻撃発動 |

### CC（群衆制御）タグ

| タグ | 効果仕様 |
|------|----------|
| スタン | 短時間の行動停止を与える |
| 恐怖 | 敵を後退させる |
| ノックバック | 敵を後ろへ吹き飛ばし位置をずらす |
| 吸引 | 敵を1点へ寄せ集める |

### 防御・回復タグ

| タグ | 効果仕様 |
|------|----------|
| 浮遊 | 近接攻撃ユニットを無視する（遠距離ユニットがいなければ近距離ユニットを狙う） |
| 召喚 | ユニットを生成する |
| 回復 | HPを回復する |
| 再生 | HPを毎秒回復する |
| ステルス | 他の味方が全滅するまで敵に狙われない |
| 回避 | 一定確率で攻撃を無効化する（上限は99） |
| 反射 | 受けたダメージの一部を相手へ返す |
| 追い風 | スタック量分、攻撃速度UP% |
| 鋼鉄 | 鋼鉄のスタック量分、自身の受けるダメージを定数ダウン |
| 吸血 | 与ダメージの一部をHPとして吸い取る |

### トリガー条件タグ

| タグ | 効果仕様 |
|------|----------|
| 着弾 | 攻撃が敵にHITした時 |
| 被弾 | 攻撃が自身にHITした時 |

## 5. 実装にあたっての技術指針

### 新規 PassiveAbility type の追加

上記の効果を実現するために必要な新しい PassiveAbility type を追加してください。
既存のtype一覧（entities.ts参照）で足りない分のみ新規追加してください。
新規typeを追加する際は:
- PassiveAbility の type ユニオン型にリテラルを追加
- PASSIVE_PHRASE_TAGS にタグマッピングを追加
- ゲームループ内に発動ロジックを実装

### EntityState への追加

スタック制の状態異常には、EntityState に対応するフィールドを追加してください:
- 出血用: bleedStacks?: number
- 炎上用: burnStacks?: number
- 凍結用: freezeStacks?: number
- 呪い用: curseStacks?: number
- 追い風用: tailwindStacks?: number
- 鋼鉄用: ironStacks?: number
- 充填用: chargeStacks?: number
- 恐怖用: fearFrames?: number

### 状態異常のtick処理

ゲームループ内で以下を処理してください:

- 出血: 被ダメージ時に bleedStacks 分だけダメージを加算。スタックは自然減衰しない
- 毒: 毎秒判定。スタック量が現HP値を上回るとスタック分の固定ダメージ。スタック制に移行
- 炎上: 0.5秒ごとに burnStacks 分のダメージ。ダメージ発生ごとに burnStacks = Math.floor(burnStacks / 2)。0で消滅
- 凍結: freezeStacks に応じて移動速度と攻撃速度をスタック量%低下。毎秒1ずつ自然減衰
- 呪い: curseStacks に応じて被ダメージを割合増加。毎秒1ずつ自然減衰
- 恐怖: fearFrames > 0 の間、移動方向を反転して後退。攻撃不可（stunと同様だが移動はする）

### 複合効果の処理

- 「クリティカルX%:○○」のような複合効果は、ヒット時にX%の確率判定を行い、成功時にもう片方の効果を発動するロジックをゲームループ側に実装
- 「充填:○○」は0.1秒（6フレーム@60fps）で1スタック獲得し、スタック量に応じて固有効果を強化
- 「着弾:○○」は攻撃HIT時にトリガーし、指定の効果を発動
- 「被弾:○○」は自身がHITされた時にトリガーし、指定の効果を発動
- 「死亡時:○○」は自身の死亡時にトリガーし、指定の効果を発動

### phrases.ts への同期

新規追加する PassiveAbility type は必ず PASSIVE_PHRASE_TAGS にエントリを追加してください。
PhraseTrigger, PhraseEffect, PhraseStatus, PhraseBuff の型定義にも不足分を追加してください。

## 6. 実装の優先順位

1. EntityStateの拡張 — 新規スタック系フィールド追加
2. 状態異常tick処理 — ゲームループに出血・炎上・凍結・呪い・恐怖の処理を追加
3. 新規PassiveAbilityの実装 — 設計書のタグを実現するために不足しているtypeを追加
4. 全バリアントの効果更新 — UNIT_STATS の passiveAbilities を設計書に合わせて更新
5. phrases.ts の同期 — 新規typeのタグマッピング追加
6. レリックの実装 — 賢者の石、火薬割引券を追加

---

## 7. 注意事項

- value の具体的な数値は、既存ユニットのDPS目安（entities.tsのコメント参照）とバランスを取って適切な値を設定してください
- トークンユニット（ゾンビ等）の召喚ロジックは既存の SUMMON 処理を踏襲してください
- 設計書に記載のない魔物やレジェンド魔物は実装しないでください
- 各バリアントの効果は設計書の記載を正とし、既存実装と異なる場合は設計書に合わせてください
