# 実装指示：パッシブ効果タグシステムの完全実装

あなたはローグライク×タワーディフェンスゲーム「Dark-Lord」の開発を担当するコーディングAIです。
以下の設計ドキュメントに基づいて、パッシブ効果タグシステムとそれに紐づく全魔物バリアントの効果を実装してください。

---

## 1. 既存コードベースの構造

本プロジェクトは React + TypeScript + Vite で構築されています。
ゲームロジックは src/game/ 配下の以下3ファイルに集約されています。

### src/game/entities.ts

    export interface PassiveAbility {
      type: string;      // パッシブの種類（例: 'REFLECT', 'POISON' 等）
      value?: number;    // 効果量（ダメージ、割合、スタック数など）
      range?: number;    // 効果範囲（px）
      cooldown?: number; // クールダウン（frames）
    }

    export interface EntityState {
      // ...基本ステータス省略...
      passiveAbilities?: PassiveAbility[];
      // ランタイムフィールド（状態異常スタック等）
      poisonedFrames?: number;
      stunFrames?: number;
      stealthActive?: boolean;
      // ...他の一時フィールド...
    }

ユニットごとのステータスは UNIT_STATS に定義されており、passiveAbilities 配列で効果を持たせます。

### src/game/phrases.ts

定型句タグシステム。パッシブアビリティをトリガー・効果・対象・状態のタグに分解し、レリック判定に使います。

    export const PASSIVE_PHRASE_TAGS: Record<string, Phrase[]> = {
      'REFLECT': ['被弾時', '反射', '攻撃者'],
      'POISON':  ['命中時', '付与', '毒', '対象'],
      // ...
    };

### src/game/config.ts

レシピ定義（ALL_RECIPES）、レリック定義（RELICS）、盤面定数など。
各レシピの resultMap でワイルドスロットの素材に応じたバリアントIDを指定します。

## 2. タグ一覧と効果仕様

以下のタグを **すべて** ゲームロジックとして実装してください。
既に実装済みのものは仕様と一致しているか確認し、未実装のものは新規追加してください。

### 攻撃形状タグ

| タグ | PassiveAbility type | 効果仕様 |
|------|-------------------|----------|
| 爆発 | 対応する複数type（後述） | 円形の単発範囲攻撃を発生させる。value=ダメージ、range=半径 |
| 瘴気 | AREA_DOT | 円形の連続ダメージ領域を作る。value=tick毎ダメージ、range=半径 |
| 跳弾 | BOUNCE_SHOT | 攻撃が近くの別敵へ連鎖する。value=連鎖回数 |
| 貫通 | PIERCING | 弾が直線上の複数体をまとめて貫く。value=貫通距離(px) |
| 発射 | 未実装→新規追加 SPREAD_SHOT | 投射物を10°扇状に複数放つ。value=発射数 |
| レーザー | 未実装→新規追加 LASER | 一定時間残る多段ヒット攻撃。value=tick回数、cooldown=持続フレーム |

### 状態異常タグ（スタック制）

| タグ | PassiveAbility type | 効果仕様 |
|------|-------------------|----------|
| 出血 | 未実装→新規追加 BLEED | スタック量だけ被ダメージが定数増加。EntityStateに bleedStacks?: number を追加 |
| 毒 | POISON | スタック量が現HP割合を上回るとスタック分の固定ダメージ。value=付与スタック数 |
| 炎上 | 未実装→新規追加 BURN | スタック量のDoTダメージ。ドットダメージ発生ごとにスタック半減。EntityStateに burnStacks?: number を追加 |
| 凍結 | 未実装→新規追加 FREEZE | 移動速度と攻撃速度（cooldown）を低下させる。value=スタック付与数。EntityStateに freezeStacks?: number を追加 |
| 呪い | 未実装→新規追加 CURSE | 被ダメージを割合増加。value=付与スタック数。EntityStateに curseStacks?: number を追加 |

### 特殊効果タグ

| タグ | PassiveAbility type | 効果仕様 |
|------|-------------------|----------|
| クリティカル | 未実装→新規追加 CRITICAL | ヒット時に確率判定。発動時に特殊効果を起こす。value=発動確率(0-100) |
| 充填 | 未実装→新規追加 CHARGE_UP | 0.1秒(6フレーム)で1スタック獲得。スタック量に応じた固有効果を発動。EntityStateに chargeStacks?: number を追加 |
| 追撃 | 未実装→新規追加 PURSUIT | 条件発動時、クールダウンを無視して即座に攻撃を発動 |

### CC（群衆制御）タグ

| タグ | PassiveAbility type | 効果仕様 |
|------|-------------------|----------|
| スタン | ON_DEATH_STUN 等 | 短時間の行動停止。value=フレーム数。既存の stunFrames を使用 |
| 恐怖 | 未実装→新規追加 FEAR | 敵を一定時間後退させる。value=持続フレーム数。EntityStateに fearFrames?: number を追加 |
| ノックバック | KNOCKBACK | 敵を後方へ吹き飛ばす。value=ノックバック距離(px) |
| 吸引 | 未実装→新規追加 PULL | 敵を1点へ寄せ集める。value=吸引力、range=効果範囲 |

### 防御・回復タグ

| タグ | PassiveAbility type | 効果仕様 |
|------|-------------------|----------|
| 浮遊 | 未実装→新規追加 FLOAT | 近接攻撃ユニットに狙われない（遠距離がいない場合は近距離を狙う） |
| 召喚 | SUMMON, CLONE 等 | ユニットを生成する。既存の召喚系typeを統合管理 |
| 回復 | HEAL_SHOT, HEAL_AURA 等 | HPを回復する |
| 再生 | SELF_REGEN, AURA_REGEN | HPを毎秒回復する。value=回復量 |
| ステルス | UNTARGETABLE, STEALTH | 他の味方が全滅するまで敵に狙われない |
| 回避 | EVADE | 一定確率で攻撃を無効化（上限99%）。value=回避率(0-1) |
| 反射 | REFLECT | 受けたダメージの一部を相手へ返す。value=反射割合(0-1) |
| 追い風 | 未実装→新規追加 TAILWIND | スタック量分、攻撃速度UP%。EntityStateに tailwindStacks?: number を追加 |
| 鋼鉄 | 未実装→新規追加 IRON | スタック量分、受けるダメージを定数ダウン。EntityStateに ironStacks?: number を追加 |
| 吸血 | LIFESTEAL | 与ダメージの一部をHPとして吸収。value=吸血割合(0-1) |

### トリガー条件タグ

| タグ | 対応するPhraseTrigger | 説明 |
|------|---------------------|------|
| 着弾 | 命中時 | 攻撃が敵にHITした時に発動 |
| 被弾 | 被弾時 / 被攻撃時 | 攻撃が自身にHITした時に発動 |

## 6. 状態異常のtick処理仕様

ゲームループ内で毎フレーム（または一定間隔で）以下を処理してください：

### 出血
- 被ダメージ時に bleedStacks 分だけダメージを加算する
- スタックは時間経過で自然減衰しない（追加付与で上書き、または専用のクリア手段）

### 毒
- 毎秒判定：poisonStacks > currentHp の一定割合 の場合、スタック数分の固定ダメージ
- ※既存の poisonedFrames とは別管理。スタック制に移行

### 炎上
- 0.5秒ごとに burnStacks 分のダメージを与える
- ダメージ発生ごとに burnStacks = Math.floor(burnStacks / 2)
- スタック0で消滅

### 凍結
- freezeStacks に応じて移動速度と攻撃速度（maxCooldown）をN%低下
- 計算例：slowRate = Math.min(freezeStacks * 0.02, 0.8)  (上限80%減速)
- スタックは毎秒1ずつ自然減衰

### 呪い
- curseStacks に応じて被ダメージを割合増加
- 計算例：damageMultiplier = 1 + curseStacks * 0.05
- スタックは毎秒1ずつ自然減衰

### 恐怖
- fearFrames > 0 の間、敵は現在の移動方向を反転して後退する
- 攻撃不可（stunと同様だが移動はする）

## 7. 実装の優先順位

1. **EntityStateの拡張** — 新規フィールド追加
2. **状態異常tick処理** — ゲームループに出血・炎上・凍結・呪い・恐怖の処理を追加
3. **新規PassiveAbilityの実装** — BLEED, BURN, FREEZE, CURSE, CRITICAL, CHARGE_UP, PURSUIT, FEAR, PULL, FLOAT, TAILWIND, IRON, SPREAD_SHOT, LASER, SUMMON_BOOST
4. **既存バリアントの効果更新** — UNIT_STATS の passiveAbilities を設計書に合わせて更新
5. **phrases.ts の同期** — 新規typeのタグマッピング追加
6. **レジェンドレシピの追加** — config.ts の ALL_RECIPES にアライブを追加

## 8. 注意事項

- value の具体的な数値が「回復量」「ダメージ」等の仮置きになっている箇所は、既存ユニットのDPS目安（entities.tsのコメント参照）とバランスを取って適切な値を設定してください
- 「クリティカル＋別効果」のような複合パッシブは、CRITICAL のヒット判定成功時にもう片方の効果を発動するロジックをゲームループ側に実装してください
- 「充填」は時間経過でスタックが溜まる仕組みのため、バトル開始からのフレーム数でスタックを計算してください（0.1秒 = 6フレーム @60fps で1スタック）
- 新規typeを追加する際は PassiveAbility の type ユニオン型にもリテラルを追加してください
- トークンユニット（ゾンビ等）の召喚ロジックは既存の SUMMON 処理を踏襲してください
