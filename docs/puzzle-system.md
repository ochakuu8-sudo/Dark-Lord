# ダークロード パズルシステム説明書

他のAIとレシピ設計を相談するための資料です。

---

## ゲーム概要

「ダークロード」は**スマートフォン横持ち**を前提としたパズル×タワーディフェンスゲームです。
プレイヤーは魔王として、パズルボードで素材を並べて魔物を召喚し、押し寄せる英雄軍を撃退します。

---

## ゲームの流れ

1. **儀式フェーズ**（パズルボード）でピースを並べて魔物を召喚する
2. **防衛フェーズ**で召喚した魔物が自動で戦う
3. 撃退成功 → 翌日へ。失敗（拠点HPゼロ）→ ゲームオーバー

---

## パズルボード

- **9×9マスのグリッド**
- 毎ターン開始時に一定数のピースがランダムに降ってくる
- プレイヤーはピースをドラッグして自由に並び替えられる
- 儀式実行ボタンを押すと、盤面に成立しているレシピをまとめてチェックし、マッチした分だけ魔物を召喚する

---

## 素材（ピースの種類）

現在3種類の基本素材があります。

| 素材 | 記号 | 絵文字 | 特性イメージ |
|------|------|--------|-------------|
| 骨 | 0 | 🦴 | 耐久・防御系 |
| 肉 | 1 | 🥩 | 攻撃・近接系 |
| 霊 | 2 | 🔮 | 魔法・遠距離系 |
| ワイルド | 9 | ❓ | 後述 |

ピースの出現率は、**装備中のレシピで使われている素材の比率**に依存します。
例：肉2個+ワイルド1個のレシピだけ装備 → 肉が多めに出る。

---

## レシピシステム

レシピとは「ボード上の特定の形に素材が揃うと魔物が召喚される」というルールです。

### レシピの構成

```
pattern: ボード上の形（マス配置）
  各セルの値: 0=骨, 1=肉, 2=霊, 9=ワイルド, -1=空白（形の穴）
```

### マッチング

- パターンがボード上のどこかに一致していたら召喚発動
- **同じレシピが複数箇所に成立していれば複数体召喚される**
- 複数の異なるレシピが同時に成立していれば全部まとめて召喚される

### ワイルドスロット（❓）

各レシピには **ワイルドスロットが1個** あります。
このスロットには骨・肉・霊のどれが入っても成立します。
**何が入ったかによって召喚される魔物の種類（派生）が変わります。**

```
例: オーク（肉肉/肉❓）の場合
  ❓に骨が入った → orc_bone（骨派生オーク）召喚
  ❓に肉が入った → orc_meat（肉派生オーク）召喚
  ❓に霊が入った → orc_spirit（霊派生オーク）召喚
```

同じ形・同じ素材でも、ワイルドに何が入るかでステータスや特性が異なるユニットになります。

## 隣接ボーナス

召喚されたユニットのマスの上下左右に、**そのユニットと同じ素材のピースが置かれていると強化**されます。
隣接1マスにつき HP・攻撃力が +15% されます（最大4マス分）。

---

## ユニットの派生（バリアント）

各レシピにつき3種類の派生があります。骨・肉・霊のどれがワイルドに入ったかで決まります。

**一般的な傾向：**
- 骨派生 → HP高め・耐久寄り
- 肉派生 → 攻撃力高め・DPS寄り
- 霊派生 → 射程が長い・魔法寄り

ただしこれは目安で、レシピごとに大きく異なります。

---

## ユニットのタグ（属性）

各ユニットは以下のタグを持ちます。パッシブ効果の設計に使います。

| タグ | 値 | 説明 |
|------|---|------|
| materialType | 0/1/2 | ワイルドに入った素材（骨/肉/霊） |
| attackType | melee/ranged | 近接か遠距離か |
| size | ピクセル半径（10〜26） | ユニットの大きさ |
| accuracy | 0〜1 | 命中率（現在全員1.0） |

---

## パッシブ能力（現在設計中）

現在全ユニットのパッシブは一旦取り外し中です。
今後、各ユニットの派生ごとに1〜2個のパッシブを付ける予定です。

実装済みのパッシブ型（再利用可能）：

| ID | 効果 |
|----|------|
| REFLECT | 被ダメージの一部を攻撃者に反射 |
| AURA_REGEN | 周囲の味方HPを毎秒回復 |
| PIERCING | 攻撃が敵を貫通 |
| PROXIMITY_EXPLOSION | 敵が近づくと自爆・範囲ダメージ |
| ATK_BUFF | 周囲の味方の攻撃力を上昇 |
| HEAL_SHOT | 最も傷ついた味方を回復 |
| SUMMON | 一定時間ごとにゾンビ召喚 |
| CORPSE_EXPLOSION | 近くの味方の死で爆発 |
| INSTANT_AOE | 敵の位置に直接範囲魔法発動 |

## レア度システム

| レア度 | 素材数の目安 | 入手方法 |
|--------|------------|---------|
| common | ワイルド除き2〜3個 | ゲーム開始時にランダム3種付与 |
| rare | ワイルド除き4〜5個 | ドラフト報酬 |
| legend | ワイルド除き5〜6個 | 未実装 |

ゲーム開始時はコモンレシピから骨・肉・霊それぞれ1種類ずつランダムに3枚装備した状態でスタートします。
2日目以降は戦闘後のドラフトで新しいレシピやレリックを獲得できます。

---

## 設計上の制約・注意点

- **パターンの重複問題**: 複数のレシピが似た形・素材だと、一方の召喚を狙うつもりが別のレシピも巻き込んで召喚されてしまいます。特に霊ウィザード（十字形）の周囲に霊を置くと意図せずマッチが増える問題が過去に発生しました。
- **ワイルドスロットは1個のみ**: 現在の仕様ではすべてのレシピにワイルドが正確に1個あります。
- **ボードサイズは9×9**: 大きすぎるパターン（5×5など）は実用的でない可能性があります。
- **出現率と素材数のバランス**: 素材数の多いレシピを複数装備すると、盤面がその素材に偏りすぎて他のレシピが成立しにくくなることがあります。
